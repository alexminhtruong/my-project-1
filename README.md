# Inlämningsuppgift

Här är repot för din inlämningsuppgift.

Om filerna:

- `.gitignore` - Denna fil används för att ignorera/exkludera vissa systemfiler som inte behöver komma med i versionshanteringen. En sådan bör man alltid ha i sitt repo.
- `index.html` - För att sidan ska kunna publiceras automatiskt på GitHub Pages är det viktigt att filen heter `index.html` med ett litet `i` i början.
- I mappen `.github` finns ett script som automatiskt publicerar sidan till GitHub Pages.
- I mappen `.github` finns även instruktioner för hur CoPilot ska ge feedback på koden.
- Det går bra att redigera denna README-fil.



# 1-steget
<p> Jag startade med att tänka på hur jag ville att min design ska se ut och fick ett hum om hur jag ville göra det.<p>

## 2-steget
<p>När jag skulle göra en wireframe för min layout hade jag det i huvudet och inte nedskrivet till en början. <br> Jag hade redan börjat att skriva min html och när jag hade fått ihop den strukturen jag ville ha pausade jag med html och gick till att styla.</p>

### 3-steget
<p> Efter att jag hade stylat de html-element som existerade så som jag ville ha det rent estetiskt började jag att lägga till de extra tilläggen enligt kravlistan:
- Navigering
- Kontaktuppgifter
- Dropdownbutton
- Formuläret
- Lägger till bilder som behövs.</p>

#### 4-steget
<p> Gå igenom html dokumentet och debuggar alla fel som dök upp pga. de nya implementeringar och stylar dem i CSS.</p>


##### 5-steget
<p>Korrigerar alla bilformat från png till webp och börjar att kontrollera i W3C Validering for html och a11y innan inlämning. Fixar även till designen rent estetiskt för att responsiviteten ska bli bättre.</p>


###### 6-steget
<p>Fixar alla felmeddelanden och stylar om text-färgerna för att öka a11y som t.ex kontrasten, mellanrum mellan de olika projekt-elementen.<br>
Lägger till sidobar som jag hade glömt och ändra storleken för headerprofile så den blir mindre och tar mindre utrymme.
</p>
